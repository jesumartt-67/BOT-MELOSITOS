**Objetivo del proyecto:**
Desarrollar una aplicación web con arquitectura de microservicios que lea información desde PDFs (costos educativos, jornadas extensivas, uniformes, fechas importantes, actividades extracurriculares, etc.), la almacene en una base de datos vectorial y la sirva a través de un chatbot que responda consultas de clientes.

**Requerimientos técnicos:**

1. **Front-End:** Interfaz básica que permita la interacción del usuario final con el chatbot.

2. **Back-End:** Sistema de almacenamiento de información de PDFs en base de datos vectorial.

3. **API REST-ful:** Comunicación entre Front-End (ChatBot) y Back-End (BD vectorial).

4. **Estructura de carpetas para cada microservicio:**
   ```
   /MicroServicio/
   ├── /embedding-service
   ├── /ingestion-service
   ├── /query-service
   ├── /vector-store-service
   └── docker-compose.yml
   ```

5. **Microservicios a implementar:**
   - 5.1 Servicio de mensajería: Comunicación con el usuario (web o WhatsApp)
   - 5.2 Servicio de inferencia: Modelo de Machine Learning para procesar mensajes
   - 5.3 Servicio de usuario/historial: Almacena conversaciones y contexto en BD
   - 5.4 Servicio de API Gateway: Seguridad y balanceo de carga

6. **Despliegue:** Google Cloud
